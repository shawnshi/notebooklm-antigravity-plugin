## 2024-06-22 - [JSON extraction bottleneck]\n**Learning:** The use of `re.search(r'(\{.*\}|\[.*\])', text, re.DOTALL)` to extract JSON from text containing large amounts of noise (like Chromium logs) can be extremely slow due to regex backtracking and copying over large strings. String `.find()` and `.rfind()` methods are order of magnitude faster.\n**Action:** Replace regex-based JSON extraction with simple string `.find()` and `.rfind()` where possible, especially on texts containing arbitrary prefixes/suffixes.
